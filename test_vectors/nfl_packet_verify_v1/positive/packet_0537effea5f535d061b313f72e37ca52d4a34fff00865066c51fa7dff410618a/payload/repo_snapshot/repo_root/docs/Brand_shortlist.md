\## 2. Naming Rules (what we will obey)



A name is acceptable only if it satisfies:



1\) No mention of other projects, engines, or internal worlds  

2\) Does not promise intelligence (avoid “mind,” “brain,” “oracle,” “genius”)  

3\) Suggests custody, law, integrity, verification, determinism  

4\) Easy to pronounce  

5\) Short enough to say in one breath  

6\) Can plausibly be used as a domain + GitHub org name  

7\) Avoids obvious collisions with giant incumbents
